Azure Load Balancer

Azure Load Balancer is a high-performance, low-latency Layer 4 (TCP/UDP) load balancer that distributes incoming network traffic across multiple virtual machines or instances. It operates at the transport layer of the OSI model and is ideal for scenarios requiring ultra-low latency and high throughput.

Key Features

  • Layer 4 Load Balancing: Distributes traffic based on IP address and port number
  • High Availability: Supports availability zones for maximum uptime
  • Port Forwarding: Enables inbound NAT rules for direct VM access
  • Health Probes: Monitors backend instance health automatically
  • Outbound Connectivity: Provides SNAT for VMs without public IPs
  • Low Latency: Microsecond-level latency for high-performance applications

Use Cases

  • Distributing traffic to VMs within a virtual network
  • Load balancing internal applications
  • High-throughput, low-latency applications
  • Non-HTTP/HTTPS protocols (gaming, IoT, databases)

SKUs

Basic: Free tier with limited features, suitable for dev/test environments.
Standard: Production-grade with SLA, availability zones, and advanced features.

Azure Application Gateway

Application Gateway is a Layer 7 (application layer) load balancer that provides advanced HTTP/HTTPS traffic management capabilities. It acts as a web traffic load balancer with built-in Web Application Firewall (WAF) capabilities, SSL termination, and URL-based routing.

Key Features

  • Layer 7 Load Balancing: Content-based routing for HTTP/HTTPS traffic
  • SSL/TLS Termination: Offloads SSL processing from backend servers
  • Web Application Firewall (WAF): Protects against common web vulnerabilities
  • URL-Based Routing: Routes requests based on URL paths
  • Session Affinity: Cookie-based session stickiness
  • Autoscaling: Automatically scales based on traffic load
  • Redirection: HTTP to HTTPS redirection

Use Cases

  • Web applications requiring SSL termination
  • Microservices architectures with path-based routing
  • Applications needing WAF protection
  • Multi-site hosting on a single gateway

SKUs

Standard v2: Standard features with autoscaling.
WAF v2: Includes Web Application Firewall with OWASP protection.

Azure Front Door

Azure Front Door is a global, scalable entry point that uses Microsoft's global edge network to create fast, secure, and widely scalable web applications. It provides Layer 7 load balancing with SSL offload, URL-based routing, and intelligent traffic routing across multiple regions.

Key Features

  • Global Load Balancing: Routes traffic across multiple Azure regions
  • CDN Integration: Built-in content delivery network capabilities
  • WAF Protection: Azure Web Application Firewall integration
  • SSL Offload: Handles SSL/TLS termination at the edge
  • URL Rewriting: Modify URLs before routing to backends
  • Caching: Edge caching for improved performance
  • DDoS Protection: Built-in DDoS protection

Use Cases

  • Global web applications requiring low latency
  • Multi-region deployments with failover
  • Applications requiring CDN and load balancing
  • API gateways with global reach

SKUs

Classic: Legacy offering with basic features.
Standard: Modern offering with enhanced performance.
Premium: Advanced security features with managed rules.

Azure Traffic Manager

Azure Traffic Manager is a DNS-based traffic load balancer that distributes traffic globally across Azure regions. It operates at the DNS level, directing client DNS queries to the most appropriate service endpoint based on routing methods and endpoint health.

Key Features

  • DNS-Based Routing: Routes traffic at the DNS level
  • Multiple Routing Methods: Priority, weighted, performance, geographic, multivalue, subnet
  • Endpoint Monitoring: Health checks for automatic failover
  • Global Reach: Works with any internet-facing service
  • Nested Profiles: Combine routing methods for complex scenarios
  • Fast Failover: Quick detection and recovery from failures

Use Cases

  • Global disaster recovery and business continuity
  • Geographic traffic distribution
  • Hybrid cloud and multi-cloud scenarios
  • Low-cost global load balancing

Routing Methods

Priority: Primary/secondary failover pattern.
Weighted: Distribute traffic proportionally.
Performance: Route to closest endpoint by latency.
Geographic: Route based on geographic location.
Multivalue: Return multiple healthy endpoints.
Subnet: Route based on source IP subnet.
